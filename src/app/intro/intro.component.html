1. Creación y Actualización de un Signal

¿Qué es un Signal?

Un Signal es una variable reactiva en Angular que almacena y gestiona valores de estado de manera eficiente, permitiendo actualizaciones reactivas sin necesidad de suscripciones explícitas.

Arquitectura

  • Encapsula estado local y global de forma reactiva
  • Evita la necesidad de Subjects o BehaviorSubjects en muchos casos
  • Se puede usar en componentes, servicios o módulos compartidos

Características Clave de los Signals

  • Reactivos → Se actualizan automáticamente cuando su valor cambia
  • Síncronos → No necesitan async o subscribe()
  • Inmutables → Solo se actualizan con set() o update()
  • Eficientes → No disparan cambios innecesarios, optimizando rendimiento

Ejemplo:

Counter: {{ counter() }}

2. Uso de computed() y effect()

computed(): Valores Derivados

Sirve para calcular valores de manera automática cada vez que un Signal dependiente cambia.

¿Cuándo usar computed()?

  • Para valores calculados que dependen de uno o más Signals
  • Evita cálculos innecesarios y mejora el rendimiento

Ejemplo:

Doubled: {{ doubled() }}

effect(): Reacciones Automáticas

Se usa para ejecutar efectos secundarios cuando un Signal cambia (por ejemplo, llamadas a APIs, logs, actualizaciones en el DOM).

¿Cuándo usar effect()?

  • Para ejecutar código en respuesta a cambios en Signals
  • Ideal para eventos, sincronización con el DOM y peticiones HTTP
Ejemplo:

Cambiando de color al ser mayor que 10

3. Interoperabilidad con RxJS

Signals y RxJS pueden trabajar juntos utilizando conversión mutua:

De Signal a Observable

Se convierte un Signal en un Observable usando computed() o effect():

Ejemplo:

Valor del observable: {{ counter$ | async }}

De Observable a Signal

Convierte un Observable en un Signal reactivo:

Ejemplo:

Valor del signal timerSignal: {{ timerSignal() }}

¿Cuándo integrar con RxJS?
  • Cuando trabajamos con flujos de datos asíncronos
  • Para integrar Signals con HttpClient, interval(), WebSockets, etc.
